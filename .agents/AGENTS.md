# Workspace Rules & Instructions
## Synchronization & Cross-Device Development
- **CRITICAL**: Always run a `git pull` or execute `./pull.ps1` at the very beginning of every session/task BEFORE researching or modifying any files. This ensures the workspace is fully in sync with the latest commits from other devices.
- Always commit and push changes directly to GitHub after completing a set of features to keep the code repository live and up to date.
